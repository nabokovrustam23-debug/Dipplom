Разработанная архитектура развертывания предполагает разделение на три логических уровня, каждый из которых размещается на отдельном физическом или виртуальном узле:

1. **Клиентская часть** представляет собой пользовательское оборудование, с помощью которого осуществляется доступ к CRM-системе «Тихий час». В качестве клиентского устройства может выступать стационарный персональный компьютер, ноутбук, планшет или смартфон. На клиентском устройстве функционирует веб-браузер, который обеспечивает обработку HTML-кода, применение CSS-стилей и выполнение JavaScript-скриптов. В процессе работы системы на клиентский узел передаются:
   – HTML-страницы, являющиеся результатом рендеринга серверных представлений (Razor Pages) и содержащие разметку пользовательского интерфейса;
   – CSS-файлы стилей, обеспечивающие визуальное оформление веб-страниц;
   – файлы изображений, включающие фотографии мастеров, элементы интерьера филиалов барбершопа и компоненты интерфейса.

2. **Веб-сервер приложений** является центральным вычислительным узлом, выполняющим бизнес-логику CRM-системы и обрабатывающим HTTP/HTTPS-запросы клиентов. Благодаря кроссплатформенности фреймворка, данный узел может функционировать под управлением различных операционных систем (например, Windows Server или дистрибутивов Linux, таких как Ubuntu). В качестве среды исполнения используется ASP.NET Core Runtime (версии 8.0). На веб-сервере размещаются следующие основные артефакты:
   – сборка BarbershopCrm.Web.dll, содержащая скомпилированный код пользовательского интерфейса и логику маршрутизации;
   – сборки BarbershopCrm.Infrastructure.dll и BarbershopCrm.Domain.dll, реализующие работу с данными (через Entity Framework Core) и содержащие сущности предметной области;
   – конфигурационный файл appsettings.json с настройками приложения, включая строку подключения к базе данных (DataSource=BarbershopCrm.db).

3. **Подсистема хранения данных** обеспечивает надежное хранение, целостность и безопасность всей информации CRM-системы. В отличие от классических клиент-серверных СУБД, требующих выделенного сервера, в данном проекте используется встраиваемая реляционная база данных SQLite. Она не требует развертывания отдельного серверного узла, а функционирует непосредственно в среде веб-сервера приложений. В качестве артефакта базы данных выступает:
   – локальный файл базы данных BarbershopCrm.db, физически размещенный в каталоге веб-приложения и содержащий всю структуру данных (таблицы сущностей: клиенты, мастера, услуги, записи и т.д.), а также сами данные системы.
