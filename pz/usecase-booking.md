Таблица 1 – Главный раздел сценария варианта использования «Онлайн-запись»

| Вариант использования (прецедент) | Онлайн-запись |
|---|---|
| Актеры | Клиент |
| Краткое описание | Авторизованный клиент записывается на выбранную услугу в барбершопе. |
| Цель | Создать запись на обслуживание |
| Тип | Базовый |

Таблица 2 – Сценарий успешного выполнения варианта использования «Онлайн-запись»

| Действия актеров | Отклик системы |
|---|---|
| Шаг №1. Клиент переходит на страницу записи. | Шаг №2. Система отображает список филиалов. |
| Шаг №3. Клиент выбирает филиал. | Шаг №4. Система отображает список доступных услуг в выбранном филиале. |
| Шаг №5. Клиент выбирает услугу. | Шаг №6. Система отображает календарь выбора даты и список мастеров, оказывающих выбранную услугу. |
| Шаг №7. Клиент выбирает дату. | Шаг №8. Система отображает сетку свободных временных слотов с группировкой по мастерам. |
| Шаг №9. Клиент выбирает свободный временной слот. | Шаг №10. Система отображает сводку записи с указанием филиала, мастера, услуги, даты, времени, цены и применённой скидки по программе лояльности. |
| Шаг №11. Клиент нажимает кнопку «Подтвердить запись». | Шаг №12. Система проверяет доступность выбранного слота, создаёт запись со статусом «Создана», уведомляет клиента об успешном создании записи и перенаправляет в личный кабинет для просмотра активных записей. |

Таблица 3 – Обработка исключительных ситуаций для варианта использования «Онлайн-запись»

| Действия актеров | Отклик системы |
|---|---|
| *Исключение №1. Выбранный слот занят другим клиентом.* |
| Шаг №9. Клиент выбирает свободный временной слот. | Шаг №10. Система сообщает, что выбранный слот более недоступен, и предлагает обновить сетку слотов. |
| Шаг №11. Клиент выбирает другой свободный слот. | Шаг №12. Система продолжает выполнение сценария с шага №10 успешного сценария. |
| *Исключение №2. Клиент не заполнил обязательные поля (не выбран филиал, услуга или время).* |
| Шаг №11. Клиент нажимает кнопку «Подтвердить запись». | Шаг №12. Система отображает сообщение об ошибке валидации с указанием незаполненных полей и не создаёт запись. |
| Шаг №13. Клиент заполняет обязательные поля и повторно нажимает кнопку «Подтвердить запись». | Шаг №14. Система продолжает выполнение сценария с шага №12 успешного сценария. |
