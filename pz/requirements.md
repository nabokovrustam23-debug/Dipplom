2.1 Требования к функциональным характеристикам

При запуске CRM-системы «Тихий час» пользователю должна открываться главная страница с информацией о филиалах, услугах и мастерах. Для доступа к функционалу управления требуется авторизация.

2.1.1 Авторизация и разграничение доступа

Для перехода к авторизации на главной странице должна быть предусмотрена кнопка «Войти». Страница авторизации должна содержать следующие элементы управления:
– текстовое поле для ввода адреса электронной почты;
– текстовое поле для ввода пароля с возможностью переключения видимости символов;
– кнопка для переключения видимости символов в пароле;
– кнопка «Войти»;
– ссылка для перехода к регистрации.

При вводе неверных учётных данных система должна отображать сообщение об ошибке. После успешной авторизации пользователь перенаправляется на страницу, соответствующую его роли: клиент – в личный кабинет с историей записей, мастер – на страницу расписания, администратор – в журнал заявок, владелец – на панель аналитики.

Система должна поддерживать четыре роли: «Клиент», «Мастер», «Администратор» и «Владелец». Для каждой роли должны быть определены разрешённые действия и доступные разделы интерфейса. Неавторизованным пользователям доступна главная страница, а также форма отправки заявки.

2.1.2 Онлайн-запись клиентов

Страница онлайн-записи должна быть реализована в виде одного экрана с пошаговым отображением разделов. Клиент должен последовательно пройти пять шагов: выбор филиала, выбор услуги, выбор даты и мастера, выбор времени и подтверждение записи. Переход между шагами должен осуществляться автоматически по мере заполнения данных.

На первом шаге клиенту должен отображаться список филиалов барбершопа в виде карточек с переключателем. Каждая карточка должна содержать название филиала, адрес и режим работы. Клиент выбирает филиал нажатием на карточку.

После выбора филиала отображается второй шаг – выбор услуги. Услуги должны быть представлены карточками с переключателем, содержащими название, длительность в минутах и цену. Клиент может выбрать только одну услугу.

На третьем шаге клиент выбирает дату и мастера. Должен быть доступен календарь в виде поля ввода даты и выпадающий список мастеров выбранного филиала. В списке мастеров должна быть доступна опция «Любой свободный». При изменении даты или мастера список доступных слотов должен обновляться.

На четвёртом шаге отображается сетка доступных временных слотов, сгруппированных по мастерам. Каждый слот представлен переключателем с указанием времени. После выбора слота становится доступным пятый шаг – подтверждение записи. На экране подтверждения отображается сводка: филиал, услуга, мастер, дата и время, стоимость с учётом скидки по программе лояльности. Если у клиента есть активный уровень лояльности, отображается размер скидки и итоговая сумма. Для подтверждения должна быть доступна кнопка «Подтвердить запись». При нажатии запись сохраняется в базу данных, а клиент перенаправляется в личный кабинет.

Для неавторизованных клиентов перед подтверждением записи должна быть предусмотрена возможность указать контактные данные: имя и номер телефона. Для авторизованных клиентов эти данные должны подставляться автоматически из профиля.

2.1.3 Обработка и мониторинг записей администратором

Главная страница администратора должна содержать следующие разделы: «Расписание», «Записи», «Заявки» и «Аналитика». Навигация между разделами должна осуществляться через панель вкладок.

Раздел «Расписание» должен отображать расписание записей всех мастеров на выбранную дату. Должна быть предусмотрена навигация по дням с помощью кнопок «Предыдущий день» и «Следующий день», а также кнопка «Сегодня» для возврата к текущей дате. Для владельца должна быть предусмотрена возможность переключения между филиалами с помощью выпадающего списка. Записи должны быть сгруппированы по мастерам. Для каждой записи должны отображаться время, имя клиента, перечень услуг, статус и итоговая стоимость. При двойном клике по записи должно открываться диалоговое окно с подробной информацией о записи.

Раздел «Заявки» должен содержать список заявок от незарегистрированных пользователей. Каждая заявка должна быть представлена в виде карточки с конвейером статусов. Заявка должна проходить следующие статусы: «Новая», «В работе», «Итог» (закрыта с записью, закрыта без записи или отклонена). Для новой заявки должны быть доступны кнопки «Взять в работу» и «Отклонить». Для заявки в работе – кнопки «Создать запись» (перенаправляет на страницу создания записи), «Закрыть без записи» и «Отклонить». Закрытые заявки должны отображаться в режиме просмотра с указанием результата и ссылкой на созданную запись при наличии. Должна быть предусмотрена фильтрация заявок по статусу, текстовый поиск по имени и номеру телефона, фильтрация по диапазону дат, сортировка и пагинация.

Раздел «Записи» должен обеспечивать просмотр и управление всеми записями, а также создание новых записей администратором.

Раздел «Аналитика» должен отображать статистические показатели.

Для администратора, привязанного к конкретному филиалу, в разделе «Заявки» должны отображаться только заявки, относящиеся к его филиалу. Владелец видит данные по всем филиалам.

2.1.4 Просмотр расписания и управление статусами записей мастером

Страница мастера должна содержать раздел «Моё расписание» с расписанием записей на выбранную дату. Должна быть предусмотрена навигация по дням с помощью кнопок «Предыдущий день» и «Следующий день», а также кнопка «Сегодня» для возврата к текущей дате.

В расписании должны отображаться все записи мастера на выбранную дату. Каждая запись должна содержать время, имя клиента, перечень услуг, их длительность и стоимость, а также текущий статус. Мастер должен иметь возможность переводить запись между статусами: «Подтверждена», «Выполняется», «Завершена». Для этого у каждой записи должна быть предусмотрена кнопка «Начать» для перевода в статус «Выполняется» и кнопка «Завершить» для перевода в статус «Завершена». Если клиент не явился на запись, мастер должен иметь возможность отметить запись как «Неявка».

Мастер должен иметь доступ к истории своих записей с возможностью просмотра завершённых и отменённых записей за прошедшие даты.

2.1.5 Автоматический расчёт стоимости услуг с учётом программы лояльности

Система должна автоматически рассчитывать итоговую стоимость записи на основе выбранной услуги и программы лояльности клиента. Для каждого клиента должна храниться информация о его уровне лояльности. Система должна поддерживать три уровня: «Стандарт» (до 10 посещений), «VIP» (от 10 посещений) и «Премиум» (от 50 посещений). Каждому уровню должен соответствовать определённый процент скидки: «Стандарт» – 0%, «VIP» – 10%, «Премиум» – 15%.

При формировании записи система должна определять текущий уровень лояльности клиента, применять соответствующую скидку к стоимости выбранной услуги и отображать итоговую стоимость с учётом скидки. Информация о скидке должна отображаться на этапе подтверждения записи клиентом, а также в интерфейсах мастера и администратора.

Повышение уровня лояльности должно происходить автоматически на основе общей суммы всех завершённых записей клиента. Система должна отслеживать накопленную сумму и пересчитывать уровень лояльности после каждого завершённого визита.

2.1.6 Формирование аналитической отчётности для владельца

Панель управления владельца должна содержать следующие разделы: «Филиалы», «Услуги», «Пользователи», а также вкладки «Расписание», «Записи», «Заявки» и «Аналитика».

Раздел «Аналитика» должен отображать ключевые показатели деятельности сети барбершопов: количество записей, средний чек, общую выручку. Показатели должны отображаться за выбранный период. Должен быть предусмотрен график выручки по дням.

Раздел «Клиенты» (в составе аналитики) должен отображать список клиентов с указанием уровня лояльности, количества посещений и общей суммы затрат. Должна быть предусмотрена группировка клиентов по уровням лояльности с отображением количества клиентов в каждой группе.

Раздел «Филиалы» должен обеспечивать управление филиалами сети. Раздел «Услуги» – управление каталогом услуг. Раздел «Пользователи» – управление учётными записями пользователей и их ролями.

2.1.7 Обработка заявок от незарегистрированных пользователей

На главной странице сайта для неавторизованных пользователей должна быть доступна кнопка «Оставить заявку». При нажатии на кнопку должна открываться страница с формой заявки.

Форма заявки должна содержать следующие элементы управления:
– выпадающий список для выбора филиала;
– выпадающий список для выбора услуги;
– текстовое поле для ввода имени;
– текстовое поле для ввода номера телефона;
– кнопка «Отправить заявку».

После отправки заявки система должна сохранять её в базу данных и отображать сообщение об успешной отправке. Новая заявка должна появляться в разделе администратора «Заявки» со статусом «Новая».

2.1.8 Выход из учётной записи

Для выхода из учётной записи у всех пользователей независимо от роли должна быть предусмотрена кнопка «Выйти», расположенная в шапке сайта. При нажатии на кнопку сессия пользователя должна завершаться, и должен выполняться переход на главную страницу.
