Учитывая возможности имеющегося оборудования и программного обеспечения, при разработке системы необходимо было избежать высокой стоимости внедрения и сопровождения. Специфика работы администратора и мастера предполагает постоянное взаимодействие с приложением в течение рабочего дня, что требует стабильной работы и интуитивно понятного интерфейса. Наиболее целесообразным решением стала разработка веб-приложения. Такой подход не требует установки дополнительного программного обеспечения на рабочие места персонала – достаточно браузера, что существенно упрощает развёртывание и обновление системы. Для разработки CRM-системы «Тихий час» были выбраны следующие средства разработки:

– среда Microsoft Visual Studio 2022;
– язык программирования C# и платформа ASP.NET Core 8;
– система управления базами данных SQLite (в режиме разработки) с возможностью перехода на MS SQL Server в продуктивной среде.

Среда Microsoft Visual Studio 2022 была выбрана благодаря понятному интерфейсу, автоматическому обнаружению ошибок в коде, встроенному отладчику и высокой производительности. Встроенные средства работы с ASP.NET Core и Entity Framework Core значительно ускоряют разработку веб-приложений.

Язык программирования C# был выбран как один из самых мощных и востребованных языков в корпоративной разработке. C# поддерживает наследование, перегрузку операторов, статическую типизацию и строгую типизацию. Объектно-ориентированный подход позволяет решать задачи по построению крупных, но при этом гибких и легко поддерживаемых приложений. Платформа ASP.NET Core 8, в свою очередь, обеспечивает кроссплатформенность, высокую производительность и современный конвейер обработки запросов.

Платформа ASP.NET Core была выбрана как современная, кроссплатформенная и высокопроизводительная версия ASP.NET. В отличие от классического ASP.NET Framework, ASP.NET Core работает на Windows, Linux и macOS, что даёт свободу выбора хостинга. Встроенный контейнер зависимостей (DI), middleware-конвейер и Razor Pages упрощают разработку и тестирование. Razor Pages, в отличие от MVC, позволяют создавать страницы с собственной моделью и обработчиками в одном файле, что уменьшает количество кода и упрощает навигацию по проекту.

В качестве СУБД были рассмотрены MS SQL Server, PostgreSQL и SQLite. Все три системы являются распространёнными решениями, однако имеют существенные различия, влияющие на удобство разработки и сопровождения. MS SQL Server – комплексное решение от Microsoft, тесно интегрированное со средой Visual Studio и платформой .NET. PostgreSQL – мощная объектно-реляционная СУБД с открытым исходным кодом, популярная в веб-разработке. SQLite – встраиваемая реляционная СУБД, не требующая отдельного серверного процесса. Для выбора наиболее подходящей СУБД был проведён сравнительный анализ по критериям: развёртывание, простота использования, безопасность и масштабируемость. Результаты сравнения представлены в таблице 4.

На основе проведённого сравнительного анализа для разработки CRM-системы была выбрана комбинированная стратегия: в режиме разработки и тестирования используется SQLite, что позволяет избежать затрат на серверное оборудование и упрощает процесс разработки. Для продуктивной среды предусмотрена возможность перехода на MS SQL Server, что обеспечит надёжное хранение данных и поддержку многопользовательского доступа в условиях локальной сети барбершопа.
