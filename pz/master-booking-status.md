Для работы с записями мастер использует страницу «Мои записи», доступную по адресу /Master/Bookings. На странице отображается лента записей на выбранную дату с информацией о клиенте, услуге, цене и текущем статусе.

Для каждой записи мастер может выполнить следующие действия в зависимости от её статуса:

1. **Подтверждение записи** (Confirm) – доступно для записей со статусом «Created». При нажатии кнопки «Подтвердить» статус записи изменяется с «Новая» на «Подтверждена». Метод «ConfirmAsync» в классе BookingService проверяет, что текущий статус равен «Created», и что мастер имеет право изменять данную запись (через привязку к его учётной записи). После успешного подтверждения запись перемещается в список подтверждённых и становится доступной для дальнейших действий.

2. **Завершение визита** (Complete) – доступно для записей со статусом «Created» или «Confirmed», при условии, что текущее время не ранее времени начала записи. При нажатии кнопки «Завершить визит» мастер может указать комментарий. Метод «CompleteAsync» создаёт объект Visit с суммой услуги (с учётом скидки по программе лояльности) и временем завершения, после чего статус записи изменяется на «Completed». Созданный визит учитывается в отчётах аналитики.

3. **Отметка «Не пришёл»** (NoShow) – доступно для записей со статусом «Created» или «Confirmed», при условии, что время записи уже наступило. При нажатии кнопки «Не пришёл» статус записи изменяется на «NoShow» без создания визита, что позволяет отметить несостоявшийся приём клиента.

4. **Отмена записи** (Cancel) – кнопка отображается для записей со статусом «Created» или «Confirmed» через общий компонент представления `_ScheduleDayAgenda`. Однако на странице мастера отсутствует обработчик для данной операции: при попытке отмены записи мастером будет возвращена ошибка, так как метод «OnPostCancelAsync» не реализован. Отмена записи доступна администратору через страницы «Расписание» и «Журнал записей».

Все операции, изменяющие статус записи, проходят через сервисный класс BookingService, который выполняет проверки прав доступа и корректности перехода статусов. Это гарантирует, что мастер может изменять только те записи, которые закреплены за ним, и только в допустимых состояниях.
