# Тест-кейсы интеграционного тестирования

## 1. Заявка → Запись (Lead → Booking)

**Сценарий:** Клиент оставляет заявку. Администратор создаёт запись на основе этой заявки.

**Шаги:**
1. Создать заявку (Lead) со статусом New
2. Создать запись (Booking) на ту же дату/время
3. Проверить: статус заявки изменился на Done, у заявки заполнился CreatedBookingId

**Что проверяет:** Заявка автоматически закрывается при создании записи. Связь однонаправленная (Lead → Booking).

---

## 2. Полный цикл: Заявка → Запись → Завершение → Аналитика

**Сценарий:** Полный pipeline — от заявки до отражения в отчётах аналитики.

**Шаги:**
1. Создать заявку
2. Создать запись из заявки (заявка закрывается)
3. Завершить запись (Complete) — создаётся Visit с суммой и временем
4. Проверить:
   - Visit создан, TotalAmount = PriceSnapshot (с учётом скидки)
   - SalesFunnel.Analytics: TotalLeads = 1, ConvertedToBookings = 1, Completed = 1, ConversionRate = 1.0
   - Revenue = 1500, AverageTicket = 1500

**Что проверяет:** Все звенья цепочки работают согласованно. Аналитика корректно считает конверсию и выручку.

---

## 3. Конверсия воронки с несколькими заявками

**Сценарий:** Несколько заявок с разной судьбой — проверка расчёта конверсии.

**Шаги:**
1. Создать 3 заявки:
   - Заявка А: статус InProgress (ещё не обработана)
   - Заявка Б: создана запись → Done
   - Заявка В: создана запись → Done → завершена (Completed)
2. Проверить:
   - TotalLeads = 3
   - ConvertedToBookings = 2
   - Completed = 1
   - ConversionRate = 2/3 ≈ 0.67

**Что проверяет:** В конверсию идут только заявки со статусом Done. InProgress не учитываются.

---

## 4. Изоляция аналитики между филиалами

**Сценарий:** Два независимых филиала — данные не смешиваются.

**Шаги:**
1. Создать филиал А с услугой 1500₽, филиал Б с услугой 2000₽
2. В филиале А: заявка → запись → завершение
3. В филиале Б: заявка → запись → завершение
4. Проверить:
   - Филиал А: Revenue = 1500, AverageTicket = 1500
   - Филиал Б: Revenue = 2000, AverageTicket = 2000

**Что проверяет:** Каждый филиал видит только свои данные. При запросе с BranchId = null данные не пересекаются.

---

## 5. CSV-экспорт с указанием источника (Lead)

**Сценарий:** Записи, созданные из заявок, в экспорте содержат колонку Source = "Lead".

**Шаги:**
1. Создать заявку, создать из неё запись
2. Завершить запись
3. Вызвать экспорт в CSV
4. Проверить: строка содержит Source = "Lead"

**Что проверяет:** В экспорт добавляется признак, что запись пришла через заявку, а не создана напрямую.

---

## 6. Выручка после завершения записи

**Сценарий:** После Complete у записи появляется Visit, и аналитика его учитывает.

**Шаги:**
1. Создать заказ и запись напрямую (без заявки)
2. Завершить запись через CompleteBookingCommand
3. Проверить:
   - Visit создан с корректным TotalAmount
   - Revenue в аналитике = сумме завершённых визитов
   - AverageTicket корректный

**Что проверяет:** Visit создаётся в момент Complete, а не при создании записи. Аналитика считает только завершённые визиты.
