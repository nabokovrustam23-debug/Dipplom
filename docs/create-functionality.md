# Добавлен функционал создания записей

## Обзор изменений

Добавлена возможность создания новых филиалов, услуг и мастеров на странице управления владельца.

## Новые страницы

### 1. Создание филиала
- **Путь**: `/Owner/Branches/Create`
- **Файлы**:
  - `src/BarbershopCrm.Web/Pages/Owner/Branches/Create.cshtml`
  - `src/BarbershopCrm.Web/Pages/Owner/Branches/Create.cshtml.cs`
- **Доступ**: Только для владельца (Owner)
- **Функционал**:
  - Ввод названия, адреса, телефона
  - Установка времени работы (открытие/закрытие)
  - Координаты (широта/долгота) - необязательно
  - Загрузка фото филиала
  - Установка статуса активности

### 2. Создание услуги
- **Путь**: `/Owner/Services/Create`
- **Файлы**:
  - `src/BarbershopCrm.Web/Pages/Owner/Services/Create.cshtml`
  - `src/BarbershopCrm.Web/Pages/Owner/Services/Create.cshtml.cs`
- **Доступ**: Только для владельца (Owner)
- **Функционал**:
  - Ввод названия и описания услуги
  - Установка длительности (в минутах)
  - Установка цены
  - Загрузка фото услуги
  - Установка статуса активности

### 3. Создание мастера
- **Путь**: `/Masters/Create`
- **Файлы**:
  - `src/BarbershopCrm.Web/Pages/Masters/Create.cshtml`
  - `src/BarbershopCrm.Web/Pages/Masters/Create.cshtml.cs`
- **Доступ**: Владелец (Owner) и Администратор (Admin)
- **Функционал**:
  - Ввод личных данных (ФИО, телефон, email, пол)
  - Выбор филиала
  - Ввод биографии
  - Выбор услуг, которые оказывает мастер
  - Установка статуса активности
- **Особенности**:
  - Администратор может создавать мастеров только для своего филиала
  - Автоматически создается связанная запись Persona
  - Должность по умолчанию: "Барбер"
  - Дата найма устанавливается на текущую дату

## Изменения в существующих страницах

### Страница списка филиалов (`/Owner/Branches/Index`)
- Добавлена кнопка "Создать филиал" в заголовке

### Страница списка услуг (`/Owner/Services/Index`)
- Добавлена кнопка "Создать услугу" в заголовке

### Страница управления мастерами (`/Masters/Manage`)
- Добавлена кнопка "Добавить мастера" в заголовке

## Валидация

Все формы включают валидацию на стороне клиента и сервера:

### Филиалы
- Название: 2-120 символов (обязательно)
- Адрес: 5-300 символов (обязательно)
- Телефон: формат российского номера (необязательно)
- Время работы: формат ЧЧ:ММ (обязательно)
- Координаты: широта (-90 до 90), долгота (-180 до 180)
- Фото: максимум 5 МБ

### Услуги
- Название: 2-120 символов (обязательно)
- Описание: максимум 500 символов (необязательно)
- Длительность: 5-480 минут (обязательно)
- Цена: 0.01-999999.99 (обязательно)
- Фото: максимум 5 МБ

### Мастера
- Фамилия: 1-60 символов (обязательно)
- Имя: 1-60 символов (обязательно)
- Отчество: максимум 60 символов (необязательно)
- Телефон: формат российского номера (обязательно)
- Email: корректный email (необязательно)
- Пол: М или Ж (обязательно)
- Филиал: выбор из списка (обязательно)
- Биография: максимум 500 символов (необязательно)

## Уведомления

После успешного создания записи:
- Отображается сообщение об успехе через TempData
- Происходит редирект на страницу списка соответствующей сущности

## Дизайн

Все новые страницы используют существующую дизайн-систему:
- Компонент `_ManagementTabs` для навигации
- CSS классы: `management-section`, `form-grid`, `form-group`, `btn`, `card`
- Иконки SVG в стиле Feather Icons
- Адаптивная верстка

## Безопасность

- Все страницы защищены атрибутом `[AuthorizePage]`
- Администраторы ограничены своим филиалом при создании мастеров
- Валидация входных данных на сервере
- Защита от XSS через Razor-шаблоны
- Проверка прав доступа перед сохранением
